Square Web Payments SDK

Sandbox card form → payment token (nonce)

Quick setup

  1. Replace Application ID and Location ID in the script below with your sandbox values from the Square Developer Dashboard (Credentials + Locations).
  2. Serve this file over HTTP(S) from a local server (not file://).
  3. Use Square’s sandbox test cards (e.g. success Visa 4111 1111 1111 1111 — confirm current doc for CVV/postal).
  4. The token is a one-time nonce: your backend passes it to Square’s APIs (e.g. CreatePayment); it is not stored as a permanent card ID.

Submit runs card.tokenize() and shows the token here (demo only — send to your server in production).
